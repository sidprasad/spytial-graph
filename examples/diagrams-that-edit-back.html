Diagrams that edit back

Most diagrams are dead pictures. These are live values — drag one, and it rewrites its own source.

spytial-graph · an explorable

You draw a diagram, export a PNG, and that's the end of it. The picture and the text that made it drift apart immediately; the next person to change it starts over. A spytial-graph doesn't end there. It's a small text notation — nodes, edges, and inline spatial rules — but the rendered diagram is the same object as the text, in both directions. Edit either; the other follows.

Grab it

Here's one. Drag a node — the layout flexes to keep its shape. Use + Node and + Relation, or drag from one node onto another to connect them. Watch the panel on the right: it isn't a saved copy of the source, it's re-derived from the graph after every edit.

Constraints, not coordinates

The trick to that flexing is that you never gave any coordinates. You stated relationships, and an inline rule — @orientation — says which way edges should point. SpyTial solves for positions that satisfy the rules. So the layout isn't a thing you maintain by hand; it's a thing the graph keeps true on its own.

This one is a tree: every edge points down, left edges also go left, right edges go right. Add a relation named left or right between two nodes and it places itself — no dragging into position.

It isn't a picture — it's a value

Because the diagram is reified, your code can read it. getValue() hands you the atoms and relations as plain JSON; getSource() hands back the notation. Same object, two views — edit the graph and both update live. A node's id is its stable identity (what edges reference); give it a friendlier label="…" for display, the way these people have names but the edges still wire up by id.

Take it with you

Hit copy notation on any figure above and you have plain text you can paste into a doc, a commit, an issue. Drop it back into a page as a ```spytial-graph block and it renders again; tag the block ```spytial-graph-editable and your readers get exactly what you've been using here — a diagram that edits back.
